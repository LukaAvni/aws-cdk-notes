## Explaining the Infra

Never Ever Touch: 1, Edit occasionally: 2, Most Programming Happens Here: 3

-  **3** bin: You decide what blueprints to import from lib in order to decide what your app will use.
-  **3** lib: You create blueprints here that define what resourse are, kind of like making a build and saving it as an object, then bin just pulls that object and allocates it to the app.
-  **1** node_modules: Installed dependencies, like a python venv.
-  **3** test: Where you can write tests.

-  **2** .gitignore: Standard git ignore, these files won't be pushed to github.
-  **2** .npmignore: Standard npm ingore, these files won't be part of the npm packages.
-  **2** cdk.json: Tells CDK how to call and start creating the app because it contains start command, also holds global variables.
-  **1** jest.config.js: Config for testing framework pre-built by CDK infra.
-  **1** package-lock.json: Like a frozen requirements.txt generated by pip freeze, which locks down boto3==2.14.5 along with every single sub-dependency.
-  **2** package.json: Like a standard requirements.txt where you list boto3>=2.0.
-  **1** tsconfig.json: Just the typescript compiler configurations.

## Useful commands

* `npm run build`   compile typescript to js
* `npm run watch`   watch for changes and compile
* `npm run test`    perform the jest unit tests
* `npx cdk deploy`  deploy this stack to your default AWS account/region
* `npx cdk diff`    compare deployed stack with current state
* `npx cdk synth`   emits the synthesized CloudFormation template
